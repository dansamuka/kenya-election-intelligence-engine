                                          Built for primary-source polling records and aggregate ethical scenario analysis. Phase 0 guardrails prohibit microtargeting, covert persuasion, sensitive-trait targeting, and voter suppression. Review ambiguous extractions before publication. Phase 1 adds normalized data tables. Phase 2 adds auditable source ingestion, extraction health, and review routing. Phase 3 adds descriptive weighted polling averages, uncertainty bands, momentum, and model-quality warnings. Phase 4 adds presidential threshold diagnostics, county-threshold scaffolding, and MP-seat modeling hooks. Phase 4B integrates the ward electoral database for regional swing and county-threshold scenario diagnostics with explicit caveats. Phase 5 adds assumption-transparent coalition and scenario stress tests; these are diagnostics, not forecasts. Phase 6A treats the uploaded Excel as provisional 2022 presidential baseline data and adds MRP-lite bridge estimates pending official validation; true MRP remains unavailable until crosstabs/microdata and demographic cells are added. Phase 6B adds provisional presidential forecast diagnostics, county-threshold proxy, and battleground proxy lists; these remain unvalidated and assumption-dependent. Phase 7 adds aggregate strategic intelligence diagnostics, county-priority scores, vote-target arithmetic, battleground proxy matrix, and an issue-environment scaffold; it remains aggregate-only and does not implement microtargeting or persuasion tooling. Phase 8 adds a static warehouse, schema, API contract, and SQLite analytical package. Phase 8B adds deployable DuckDB, Parquet lakehouse outputs, PostgreSQL/PostGIS schema, Docker Compose, and live FastAPI service code; a public cloud URL still requires external deployment. Phase 9 adds source provenance, model lineage, caveat registry, data-quality scorecard, and audit API endpoints; this improves transparency but is not legal certification, official IEBC validation, or proof of forecast accuracy. Phase 10 adds release-readiness gates, runbooks, acceptance tests, final gap register, and roadmap execution discipline; it makes the system deployable with caveats, not validated as a true forecast. Phase 11 adds the official-validation bridge. Phase 11B supplies official IEBC county rows and publishes comparison results; the workbook baseline fails as an official county-level presidential baseline in most counties and should be replaced or reclassified as a separate scenario/proxy dataset. Phase 12 validates the workbook voter-register geography spine. Phase 12B adds bundled reviewed IEBC county, constituency, and ward register source rows and reruns the comparison; final external certification still requires independent full-PDF extraction. Phase 13 adds an official 2022 county historical baseline, official-county-calibrated constituency proxy rows, 2022 turnout features, and explicit 2013/2017 extraction gaps. Phase 13B extracts 2017 county presidential rows, 2013 national results, and compiled 2013/2017/2022 trend rows, while keeping full 2013 county candidate-vote rows and constituency rows as explicit gaps. Phase 14 adds the MP-seat baseline scaffold. Phase 15 adds the KNBS demographic/poststratification architecture and geography-only bridge cells; true demographic MRP remains pending reviewed KNBS demographic tables, crosswalks, and poll crosstabs. Phase 16 adds the poll-crosstab extraction architecture, crosstab inventory, methodology registry, and comparability report; true MRP remains blocked until reviewed subgroup crosstab rows or microdata are supplied. Phase 17 adds an aggregate MRP-lite v2 estimator using geography-only poststratification bridge cells, national polling anchor, constituency diagnostics, and uncertainty warnings; it is not true demographic MRP. Phase 18 adds historical back-testing diagnostics and calibration-readiness gates, plus strict loaders/templates for reviewed subgroup crosstabs and true KNBS demographic cells. Phase 18B populated internal seed crosstabs/cells for pipeline testing. Phase 18C replaces those seed rows with source-backed external reviewed rows where available: TIFA May 2026 zone crosstabs and KNBS 2019 national gender cells; full KNBS constituency/sub-county demographic-cell extraction remains pending. Phase 19 implements a full-dimensional constituency demographic grid for age, gender, and urban/rural cells; the grid is model-usable as an estimated bridge but remains uncertified until full KNBS Volume III/IV table extraction is completed. Phase 20 adds the KNBS Volume III/IV certified extraction and review gate, official source registry, reviewed CSV templates, and API endpoints; in this package certified replacement remains data-pending because no reviewed KNBS extraction rows were bundled.
