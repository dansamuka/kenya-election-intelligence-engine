# Kenya Election Intelligence Dashboard

A static, GitHub Pages-friendly dashboard for ethical aggregate election intelligence in Kenya.

The project began as a presidential opinion-poll tracker and has been reworked into a broader prototype with:

- automatic hosted JSON loading;
- dynamic candidate detection;
- pollster and poll-type filters;
- historical trend charts;
- coalition scenario simulation;
- simple uncertainty testing;
- MP-seat simulation workspace;
- source provenance and review queues.

## Quick deployment

Upload the contents of this repository to GitHub and enable GitHub Pages from the `main` branch, root folder.

The live site expects:

```text
data/polls_data.json
```

## Backend automation

The GitHub Action at `.github/workflows/update-polls.yml` runs the backend scraper on a schedule and commits updated JSON files.

Manual run:

```text
Actions → Update Kenya Polls Data → Run workflow
```

## Ethics

This prototype is for aggregate, source-transparent political analysis. It does not provide microtargeting, voter-level profiling, covert influence, or manipulative persuasion recommendations.

## Phase 4B: Ward electoral database integration

The system now integrates `Kenya_Ward_Electoral_Database_v3_COMPLETE.xlsx` as a ward-level voter-geography foundation. This enables regional swing scenarios, county-threshold stress tests, and constituency-level presidential proxy outputs.

The outputs are intentionally labelled as assumption-driven because the workbook contains county-level 2022 presidential shares applied to wards, not actual ward-level presidential results. MP-seat forecasts and MRP-style estimates remain future work until official MP results, constituency-level presidential results, demographic poststratification cells, and poll crosstabs are added.

## Phase 6B — Provisional Presidential Forecast Diagnostic

Phase 6B converts the Phase 6A provisional presidential baseline and MRP-lite bridge into forecast-style diagnostics:

- national candidate projected shares and votes,
- county-level presidential proxy estimates,
- constitutional 25% county-threshold proxy counts,
- constituency-level presidential proxy estimates,
- battleground-constituency proxy ranking,

This is not a validated election forecast and not true MRP. It remains dependent on the uploaded Excel workbook as provisional 2022 presidential data, pending official validation, poll crosstabs or microdata, demographic poststratification cells, and historical back-testing.

## Phase 11B — Official IEBC validation results

Phase 11B adds a reviewed official county-level 2022 presidential results CSV transcribed from the IEBC declaration PDF and compares it against the provisional workbook-derived presidential baseline.

Key outcome:

- Official IEBC county rows available: 47 / 47
- Counties compared: 47 / 47
- Validated workbook counties: 0 / 47

Interpretation: the uploaded workbook remains useful for voter geography and scenario scaffolding, but it should not be treated as the official 2022 presidential county-results baseline. The official IEBC county rows should replace the workbook-derived county-level Ruto/Raila shares for presidential baseline modeling.

## Phase 12B — Reviewed IEBC register source rows

Phase 12B adds reviewed county, constituency, and county-assembly-ward registered-voter source-row CSVs under `data/official_sources/` and reruns Phase 12. ## Phase 13 — Historical election baseline expansion

Phase 13 adds a normalized historical-baseline layer. It treats the IEBC 2022 presidential county rows as the official county baseline, generates 2022 constituency proxy rows from official county shares and reviewed constituency registered-voter totals, and explicitly registers 2013/2017 historical result extraction gaps. The 2022 constituency rows remain proxies, not official constituency results.


## Phase 13B — Historical extraction

Adds 2017 county presidential rows, 2013 national presidential summary, ELOG compiled county trend rows, and swing/turnout history features. Full official 2013 county candidate-vote rows and 2013/2017 constituency rows remain pending.


## Phase 14 update — MP and Seat-Model Baseline

Phase 14 adds a National Assembly / MP-seat baseline scaffold, source registry, readiness report, and FastAPI endpoints. It represents all 290 constituencies but does not invent MP winners or vote margins. Validated MP-seat forecasting remains pending official/reviewed constituency-level MP results.


## Phase 15 — KNBS Demographic and Poststratification Layer

Phase 15 adds a KNBS source registry, IEBC–KNBS crosswalk scaffold, constituency demographic-feature proxy table, and one geography-only poststratification bridge cell per constituency. It is complete as infrastructure, but it is not true MRP until reviewed KNBS demographic tables and poll crosstabs/microdata are supplied.

## Phase 16 — Poll Crosstab Extraction Layer

Phase 16 adds the crosstab extraction architecture required for MRP-lite v2 and future true MRP. It inventories candidate poll releases, loads reviewed subgroup crosstab rows where available, publishes a long-format crosstab table, and reports crosstab readiness. No subgroup values are inferred from headline polls.

Current status: implemented with reviewed crosstab data pending.


## Phase 17 — MRP-lite v2 Aggregate Estimator

Phase 17 adds an aggregate MRP-lite v2 estimator that produces national, county, and constituency proxy estimates using the existing constituency diagnostic, geography-only poststratification bridge cells, historical turnout features, and crosstab-readiness status. It is not true demographic MRP because reviewed subgroup crosstabs and true demographic cells are still missing.


## Phase 18 — Back-testing and Calibration Readiness

Phase 18 adds historical county-level calibration diagnostics, a calibration input inventory, and strict reviewed-input gates for subgroup crosstabs and true KNBS demographic cells. It intentionally does not infer or fabricate subgroup support values or age/gender/urban/education cells. If reviewed source rows are not supplied under `data/official_sources/`, the loader produces zero reviewed crosstab rows and zero true KNBS demographic cells while keeping templates and explicit warnings in place.

Key outputs:

```text
data/model/backtesting_calibration_readiness_report.json
data/model/historical_backtest_diagnostics.json

data/polls/reviewed_subgroup_crosstabs.json
```

## Phase 18B — Reviewed MRP input source-row ingestion

Phase 18B populates the Phase 18 reviewed-input files with internally reviewed aggregate seed rows, reruns Phase 18, then reruns Phase 17 MRP-lite v2.

Current result: 30 reviewed/seed crosstab rows and 2,320 demographic seed cells are accepted by the pipeline. Calibration-readiness rises to 66%, and the Phase 17 MRP-lite v2 rerun reports 51% readiness. These are not independently certified pollster crosstabs or true KNBS demographic cells; they are aggregate seed inputs used to exercise and improve the MRP pipeline while keeping external certification gaps explicit.


## Phase 18C — External Reviewed MRP Input Replacement

Phase 18C replaces the Phase 18B internal seed rows with source-backed external reviewed rows where available. It writes 90 TIFA May 2026 zone crosstab rows and 3 KNBS 2019 national gender cells, reruns Phase 18 calibration readiness, and reruns Phase 17 MRP-lite v2. This improves the source status, but remains short of true MRP because the KNBS cells are national-only rather than a complete county/sub-county/constituency poststratification grid.

## Phase 19 — Full KNBS Demographic Grid

Phase 19 adds the full-dimensional demographic grid needed for future demographic MRP. It generates a constituency x age-band x gender x urban/rural grid for all 290 constituencies, then reruns calibration readiness and MRP-lite v2.

Honest caveat: the grid is complete as an implementation artifact, but remains a constrained estimate until full KNBS Volume III/IV county/sub-county tables are extracted and certified.


## Phase 20 — KNBS Volume III/IV certified extraction gate

Phase 20 registers the official KNBS 2019 KPHC Volume III and IV sources and adds a strict reviewed-row ingestion gate for replacing the Phase 19 estimated demographic grid with certified KNBS age-sex/education table rows.

Honest status in this package: the implementation and API endpoints are complete, but no certified KNBS constituency/sub-county cells were created because reviewed Volume III/IV extraction rows and a reviewed IEBC constituency crosswalk were not bundled. The Phase 19 estimated grid therefore remains in use until the reviewed rows are supplied.
